## 1. 배경 — 왜 고정 IP 가 필요한가

### 1-1. 현 상황
세정-CRM 의 문자 발송 인프라는 **Vercel Serverless** 환경에서 동작.
실제 발송 API 는 sendon (sendon.kr) 단일 벤더 사용.

### 1-2. 문제
sendon 은 **API 호출 IP whitelist 가 필수** — 등록되지 않은 IP 에서의
발송 요청은 자동 거부됨 ("등록되지 않은 IP 에서 요청되어 발송이 실패했습니다").

Vercel Serverless Function 은 AWS Lambda 풀에서 매 호출마다 다른 IP 를
할당하는 구조라 **고정 IP 1개로 묶이지 않음**. 따라서 임의 IP 를 등록해도
다음 호출에서 다시 차단되는 무한반복.

sendon 의 IP 등록은 `/28 ~ /32` CIDR 만 허용(최대 16개). Vercel 의 일본
도쿄 Lambda 풀은 수백~수천 개 IP 라 등록 자체가 불가능.

### 1-3. 영향
- 학부모 SMS 발송이 실 통신사로 전달되지 않음
- 위저드 상에는 "발송 완료" 로 표시되나 실제 미발송 (벤더 잔액·통계 불일치)
- 설명회 신청 흐름 전체가 막힘

→ **즉시 해결 필요**, 그러나 영구 해법은 비용 효율성 검토 필요.

## 2. 해법 — 2단계 전환 전략

### Phase 1 — 6월: Vercel Static IP (즉시 운영 정상화)

| 항목 | 내용 |
|---|---|
| 비용 | $100/월 (Vercel Static IP) + $20/월 (Vercel Pro) = **$120/월** |
| 셋업 시간 | 즉시 (Vercel UI 에서 Enable 클릭) |
| 코드 변경 | 없음 |
| 운영 부담 | 없음 (Vercel 이 IP 고정·관리·모니터링 모두 담당) |
| 신뢰성 | Vercel SLA 그대로. Vercel 인프라 다운 시에만 영향 |

장점: 클릭 1번이면 끝. 학원 운영 즉시 재개.
단점: 월 $100 (= 약 14만원) 의 인프라 비용이 영구적으로 부담.

### Phase 2 — 7월~: AWS Lightsail Relay (영구 운영)

| 항목 | 내용 |
|---|---|
| 비용 | $5/월 (Lightsail Seoul 인스턴스 + Static IP 포함) + $20/월 (Vercel Pro) = **$25/월** |
| 셋업 시간 | 2~3일 (VM 구축 + proxy 설정 + 검증) |
| 코드 변경 | 환경변수 1개 추가 (`HTTPS_PROXY`) — 어댑터 코드 거의 무변경 |
| 운영 부담 | VM 1대 유지보수·모니터링 (월 ~30분 점검) |
| 신뢰성 | Lightsail SLA 99.95%. 모니터링·자동재시작으로 보완 |

구조:
```
Vercel (Next.js · SSR) ─→ Lightsail VM [고정IP] ─→ sendon
```

Vercel 함수가 sendon 을 직접 호출하지 않고, **자체 보유한 고정 IP 의
중계기 VM** 을 거쳐 호출. sendon 입장에선 모든 요청이 그 1개 IP 에서
오는 걸로 보임 → IP whitelist 1개 등록으로 영구 동작.

장점: 비용 96% 절감. 한국 region(Seoul) 으로 지연시간 더 짧음.
단점: VM 다운 시 발송 중단. 모니터링 책임이 운영팀으로 옴.

### 3-6. 숨은 비용 / 변동 가능성

- **Vercel Pro $20 은 두 시나리오 공통** — 절감 가능 영역 아님 (CRM 의 SSR/캐시·자동 배포 등 핵심 기능 사용).
- **Lightsail 트래픽 초과**: 1TB/월 포함. 학원 발송 트래픽은 LMS 1건 ~1KB, 월 5만건 발송해도 ~50MB. 초과 가능성 사실상 없음.
- **환율 변동**: USD/KRW 가 1,500원 까지 오르면 절감액 약 7% 증가 (Phase 2 가 USD 표시 비용이 더 작아 환율 영향 작음).
- **AWS 가격 인하 가능성**: 과거 추세상 Lightsail 가격은 안정적. 단기 인하 없음.
- **Vercel 가격 인상 가능성**: Pro 가격은 2023~2025 동안 인상 이력 있음. Static IP $100 도 인상될 수 있음 → 절감 효과 커짐.

## 5. 위험 요소와 완화책

### 5-1. VM 다운 시 발송 중단
- **완화**: UptimeRobot 무료 플랜으로 1분 간격 ping. 다운 즉시 슬랙/SMS 알림.
- **자동 복구**: systemd 로 tinyproxy 자동 재시작 설정.
- **백업 경로**: 비상 시 Vercel Static IP 를 30분 안에 재활성 가능 (코드 무변경, env 만 토글).

### 5-2. 전환 중 일시적 차질
- **완화**: Phase 1 유지하며 Phase 2 구축. 검증 끝난 뒤 전환 → 동시 운영 기간에 발송 중단 없음.

### 5-3. AWS 비용 예측 실패
- **완화**: Lightsail 은 정액제($5 고정). EC2 와 달리 트래픽 폭증 시에도 청구 폭증 없음.

### 5-4. 보안 — proxy 노출 시 sendon 잔액 도용
- **완화**: tinyproxy Basic Auth 강제. 인증 정보는 Vercel env (암호화 저장) 만 보유. sendon 잔액은 한도 한정이라 손해 한도 자체가 작음.
