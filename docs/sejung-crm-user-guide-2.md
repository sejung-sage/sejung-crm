## 세정학원 CRM — 사용 가이드 ②

설명회 · 신청 링크 · 발송 그룹 고도화 (2026-06 업데이트분)

링크 url : https://sejung-crm.vercel.app/

> 가이드 ①(권한·학생·강좌·문자 발송·캠페인·템플릿·계정)에 이어, 이번 업데이트로 추가된 기능을 정리합니다.

---

### 설명회 (신규 메뉴)

**안내**: 강좌 중 "설명회"만 따로 모아 보고, 신청 현황을 한눈에 관리하는 화면입니다. 좌측 메뉴 **설명회**로 진입합니다. (설명회 = 강좌 등록 시 과목을 "설명회"로 지정한 건)

**종강 설명회 자동 제외**: 이미 끝난 설명회(이름이 `(종)`·`(폐)` 등으로 시작 — 아카2000에서 닫힌 회차)는 목록과 발송 선택에서 **자동으로 빠집니다.** 진행 예정인 설명회만 보입니다.

**목록 컬럼**


| 컬럼 | 설명 |
| --- | --- |
| 설명회명 | 클릭하면 상세로 이동 |
| 분원 | |
| 일시 | 신청 페이지에 설정한 행사 일시 (없으면 —) |
| 신청현황 | **아카 N · CRM M** 두 출처를 함께 표기. 정원이 있으면 진행바 + `CRM신청/정원` |
| 상태 | 공개중 / 마감 / 초안 / 미생성 |
| 발송 | "이 설명회로 발송" 바로가기 |

- **아카** = 아카2000에 수강 등록된 학생 수
- **CRM** = 우리 신청 링크로 직접 신청 완료한 수

---

### 설명회 상세

**안내**: 설명회 1건의 신청 페이지 설정 + 두 출처(아카/CRM) 명단을 한 화면에서 봅니다.

**① 공개 신청 페이지 설정**

- 상태(초안/공개중/마감) · 행사 일시 · 신청 시작/마감 · 정원 · 설명 편집
- "이 설명회로 발송" → 발송 위저드로 이동
- 페이지는 발송 시 자동 생성됨. 일정·정원을 미리 잡으려면 여기서 저장

**② 명단 2패널** — 출처를 구분해 나란히 표시

- 🔗 **CRM 신청생**: 신청 링크로 접수한 학부모. 이름 · 연락처 · 신청일시
    - 그 학생이 아카에도 등록돼 있으면 **`아카에도 등록`** 뱃지
- 🏫 **아카 등록 수강생**: 아카2000 수강 등록 학생. 이름 · 학교/학년 · 출석
    - 그 학생이 CRM 신청도 했으면 **`신청함`** 뱃지

**③ 엑셀 다운로드** — "설명회 명단" 우측 [엑셀 다운로드]

- 아카 + CRM을 **한 파일로 통합**, 학생 1명당 1행
- 컬럼: 이름 · **출처(아카 / CRM / 아카+CRM)** · 학교 · 학년 · 학부모연락처 · 출석 · CRM 신청일시
- 학부모 연락처는 권한에 따라 마스킹(master만 원문)

---

### 설명회 문자 발송 (초대링크)

**안내**: 문자 발송 > **설명회 문자** 위저드. 학생마다 **본인 전용 신청 링크**를 자동 발급해 보냅니다.

**진행 순서**

1. **설명회 선택** — 카드로 보여줄 설명회 1개 이상 선택 (종강 설명회는 목록에서 제외됨)
2. **대상 선택** — 발송 그룹 또는 강좌 학생
3. **본문 작성** — 본문에 **`{초대링크}`** 변수를 넣으면, 발송 시 학생별 고유 링크(`/s/...`)로 자동 치환
    - `{초대링크}`를 안 넣으면 본문 끝에 자동으로 링크가 붙습니다
    - 설명회 발송은 `{이름}` 변수는 사용 불가 (`{초대링크}`만 지원)
4. **테스트 발송 / 실제 발송**

**테스트 발송** (본문 작성 단계 카드)

- 본인 번호 1건으로 즉시 테스트
- **실제 동작하는 링크**가 생성됩니다 — 그 번호가 **학부모 연락처로 등록된 학생** 기준으로 발급
    - 예: 내 번호를 학부모 연락처로 가진 테스트용 학생이 있어야 링크가 만들어집니다
- 발송이 막혀 있어도(아래 "발송이 안 될 때" 참고) **테스트 링크가 카드에 함께 표시**되어 바로 클릭·복사해 확인할 수 있습니다

**재발송** — 캠페인 상세에서 학생별 [재발송]

- 재발송 시에도 `{초대링크}`가 그 학생의 실제 링크로 치환됩니다
- ⚠️ **재발송이 `{초대링크}` 글자 그대로 나가면**: 캠페인 페이지를 **새로고침(⌘+Shift+R)** 후 다시 [재발송] 하세요. (페이지를 업데이트 전에 열어둔 경우 옛 동작이 호출됨)
- 참고: 캠페인 상세의 "발송 내용 → 본문"에 보이는 `{초대링크}`는 **저장된 템플릿 원형**이라 원래 그렇게 표시됩니다. 실제 발송 문자에서만 링크로 바뀝니다

### 학부모 신청 페이지 (`/s/링크`)

**안내**: 학부모가 문자로 받은 링크로 들어오는 화면입니다. 폼 입력 없이 카드 1번 클릭으로 신청합니다.

- 상단에 **학생 전용** 안내 + 학생명/연락처
- 설명회 카드별 [설명회 신청하기] → **인앱 확인창** → [신청하기]로 접수
- 신청 완료 / 이미 신청 / 마감 / 신청 기간 외 / 종료 상태를 카드에서 바로 안내
- **본인 전용 링크** — 다른 사람에게 공유 금지 (공유 시 본인 신청과 충돌)

> 확인창은 앱 내 모달입니다. (이전의 브라우저 기본 팝업 "대화상자 숨기기" 문제 해결 — 학부모가 실수로 눌러 신청이 막히던 현상 제거)

### 발송 그룹 — 학생 검색으로 만들기

**안내**: 조건이 아니라 **학생을 직접 골라** 그룹을 만들 수 있습니다. (새 그룹 만들 때)

**만들기 방식 토글**

- **조건으로 만들기**: 기존 방식(학년/학교/지역/과목 등). 조건 미선택 시 분원 전체
- **학생 검색으로 만들기**: 이름·연락처로 검색해 직접 담기
    - **담은 학생만** 발송 대상입니다 — 조건 미선택이어도 전체가 되지 않음 (안 담으면 0명)
    - 검색 → 결과 클릭으로 추가, 칩의 X로 제거
    - 우측 미리보기는 담은 인원수로 실시간 갱신

**같은 분원 학생끼리만**

- 학생 검색은 **현재 분원** 안에서만 됩니다
- 분원을 바꾸면 담은 학생 명단이 **초기화**됩니다 (분원 섞임 방지)

**자주 막히는 부분**: "학생 검색으로 만들기"에서 아무도 안 담으면 0명입니다. 꼭 1명 이상 담아야 저장·발송됩니다.

### 수신거부 (opt-out)

**안내**: 학부모가 "그만 보내달라"고 하면 그 번호를 수신거부에 등록합니다. **학생 상세** 화면, 학부모 연락처 옆 **[수신거부 등록]** 버튼.

- **등록**: master·admin·manager 가능 (viewer 불가). 확인창 → 등록. 등록 후 그 번호로는 문자가 발송되지 않습니다.
- **해제**: master만 가능. 학생 상세에서 "수신거부됨" 옆 [해제].
- **발송 시 표시**: 수신거부 번호가 발송 대상에 섞이면, 그 건은 **발송되지 않고 캠페인에 "실패(수신거부)"로 기록**됩니다. (즉시 그룹 발송 기준 — 캠페인 실패 수·사유에 드러남)

---

### 발송이 안 될 때 (체크리스트)

문자가 "발송 완료"로 보여도 실제로 도달하지 않으면 아래를 확인하세요.

1. **sendon 발송 허용 IP** — sendon 콘솔 **API KEY 관리 → 허용 IP 관리**에 발송 서버 IP가 등록돼 있어야 합니다. (로그인 보안 인증의 허용 IP와는 **다른 메뉴**)
2. **발신번호 / 잔액** — sendon 발신번호 등록 및 충전 잔액 확인
3. **야간 광고 차단** — 광고 메시지는 21:00~08:00 자동 차단(예약은 다음날 8시 이후 발송)
4. **수신거부 / 탈퇴** — 수신거부 학부모·탈퇴 학생은 자동 제외됨
5. **재발송 `{초대링크}` 문제** — 위 "재발송" 항목 참고(새로고침 후 재시도)
