# 발송 안전성 리뷰

문자 발송 관련 코드 변경이 있을 때 머지 전에 실행. 한 번 뚫리면 금전·법적 손실로 직결되므로 별도 관문.

## 실행

qa-engineer 에이전트를 리뷰 모드로 호출:

```
qa-engineer에게 위임: "src/lib/messaging/ 및 관련 Server Action 변경분을 발송 안전성 관점에서 리뷰. 아래 가드가 우회 가능한 경로가 있는지 확인."
```

## 검증 항목

### 1. 수신거부 (unsubscribes) 필터
- [ ] 모든 발송 경로에서 `unsubscribes` 테이블 조회 후 제외
- [ ] 캐시된 수신자 리스트 사용 시에도 최종 단계 재확인
- [ ] 테스트 발송 경로도 동일하게 적용

### 2. 야간 차단 (21:00~08:00)
- [ ] 서버 시각 기준 (클라이언트 시각 신뢰 금지)
- [ ] 타임존은 Asia/Seoul 고정
- [ ] 광고성 메시지만 차단, 정보성은 허용
- [ ] 예약 발송 시 예약 시각 기준으로 검증

### 3. 필수 푸터 삽입
- [ ] 광고성 메시지 앞 `[광고]` 자동 삽입
- [ ] 본문 끝 080 무료수신거부 안내 삽입
- [ ] 이미 사용자가 넣은 경우 중복 삽입 방지

### 4. 비활성 학생 제외
- [ ] `status = '탈퇴'` 강제 제외
- [ ] 옵션으로 `수강이력자`, `신규리드` 포함/제외 선택 가능

### 5. 비용·상한
- [ ] 1회 발송 상한 (예: 10,000건) 초과 시 거부
- [ ] 예상 비용 사전 계산 및 UI 표시
- [ ] SMS(90b)/LMS(2000b) 바이트 계산 정확한지

### 6. 로그 마스킹
- [ ] 학부모 번호가 평문으로 로그에 남지 않는지 (`010-****-1234`)
- [ ] API 키·토큰이 로그/에러 메시지에 노출되지 않는지

## 출력

통과/실패 + 우회 가능 경로 발견 시 구체적 파일:라인 명시. 실패 시 해당 에이전트(backend-dev)에게 수정 요청.
